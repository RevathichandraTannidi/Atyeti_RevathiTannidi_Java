JSON Update Handler Using Java and Jackson:

The objective of this task is to build a Java program that reads a large main JSON
file containing user records and an input JSON file containing update requests,
then applies those updates correctly. The program should load both JSON files using
the Jackson library, find records in the main file based on the given ID, update
simple and nested fields as specified, and create a new record if the ID does not
already exist. After processing all update requests, the program must save the
updated data back to the main JSON file so that existing records are modified and
new records are added successfully.


Problem Statement: JSON File Update System Using Unique ID

In many applications, data is stored in JSON files containing arrays of objects. Each object in the array represents a record and contains a unique identifier (ID) along with other attributes.

The requirement is to develop a Java-based utility system that can read, update, and maintain JSON array files using the unique ID as the primary key.

The system should accept an input update request containing an ID and a set of fields that need to be updated. The program must locate the object in the JSON array with the matching ID and update the specified fields. If the ID does not exist in the JSON file, the system should create and append a new object with the given ID and provided values.

Additionally, the system should support nested JSON field updates using a path notation (e.g., address.street). After performing updates, the modified JSON data must be written back to the file so it can be reused by other components or future processes.

This utility should be designed to be generic and reusable, allowing updates on multiple JSON files with different structures, and should work even when real-time external systems or databases are not available.

Functional Requirements

The system must read a JSON file containing an array of objects.

Each object must contain a unique identifier field (id).

The system must accept update input containing:

ID

One or more fields to update.

If the ID exists:

Update the specified fields.

If the ID does not exist:

Create a new record and add it to the JSON array.

The system must support nested field updates using path notation (e.g., address.street).

After modification, the updated JSON array must be saved back to the file.

The solution should be implemented in Java using a JSON processing library such as Jackson Databind.

Expected Outcome

A reusable Java utility module capable of performing dynamic updates on JSON array files using unique identifiers, enabling structured data management even in environments without database access.
